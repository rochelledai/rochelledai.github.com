User Testing Results

Finding #1

The content did not display on screen correctly. Each person had a different screen size, and some of the images were cut off as a result. This is problematic because it makes some objects inaccessible, and the design feels cluttered or messy. Users would say, “Oh I didn’t even know this was a clickable image” because a majority of it was cut off and not visible on screen. Below is a screenshot of a user’s screen during the testing, which shows the viewport issue. The text starts to overlap the image on the left, which makes some text illegible. This ended up discouraging users from fully reading the descriptions. The very last user had a macbook and the site displayed correctly because the viewport was 1366x768px.

Finding #2

Users did not immediately recognize that clicking certain objects produces audio. First, users would click on an object, then read the description for a bit. Then, they would start to mouse around and sort of accidentally hover over the object again, after which, they notice the play button cursor that appears. Then they say, “Oh that’s how you play audio”. One user said that if the initial alert notification at the beginning of the test had not mentioned audio, they would have never known to click the object to produce sound. Another user shared the same sentiment about not realizing that certain objects have audio, but others don't.

Finding #3

Users struggled to pause the audio. Due to my difficulties with coding, the user must move the mouse out then back over the image in order to be able to click and pause the audio. Users would hover over the image, click the image to start playing the audio, then would repeatedly try to click again in order to pause the audio. However, pausing wouldn’t work, so they would move the mouse a bit then see that the cursor had changed to a pause icon and feel relieved. The current mode of playing and pausing the audio does not follow the users’ mental models and is difficult to use.

Plans

To remedy the problem of my site looking different on other laptops, I plan on changing the position and width of the elements to percentages instead of pixels. One of the users recommended using percentages because he has two monitors, and each have different pixels, so that is an unreliable mode of positioning the content.

To make the audio feature more obvious, I plan on changing the cursor of the play and pause buttons so that they are more visible when hovering over the objects. I will also add some text on the page that instructs the user to click the object to play the audio. By making the audio features more visible, people will be more aware of its existence.

I plan on rewriting my code so that the play and pause button for the audio works on click, without users having to mouse their mouse. I have been struggling a lot with this part of the project, but will try my best to find a workaround in order to create a more fluid experience.
